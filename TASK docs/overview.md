# juben_gen 升级计划 - 项目概览

## 项目定位
将现有「小说→短剧」生成项目的经验固化为方法论文档，并升级为全 Claude 驱动的端到端自动化剧本创作工具，支持多题材扩展。

## 核心信息

| 项目 | 内容 |
|------|------|
| **核心目标** | 经验沉淀 + 项目升级（补齐pipeline、提升质量、多题材、全自动化） |
| **目标用户** | 自己 + AI 协作（Claude） |
| **模型策略** | 全部使用 Claude，移除 OpenAI 依赖 |
| **关键约束** | 保持现有样例驱动约束体系、红果风格规则、格式规范 |
| **Skill形态** | 方法论文档，用于指导 Claude 协作写剧本 |

## 决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| Skill 形态 | 方法论文档 | 指导 AI 协作，非代码模板 |
| 模型策略 | 全部 Claude | 简化架构，统一体验 |
| 项目方向 | 升级现有项目 | 基础已有，无需重建 |
| 执行顺序 | 线性 1→2→3→4→5 | 经验先沉淀，再逐步升级 |
| Web UI / 版本管理 / 协同 | 不需要 | 保持 CLI 工具定位 |

## 经验沉淀重点

- 样例驱动约束：从优秀样例统计数据指标作为创作约束
- 红果风格规则：高密度冲突、快节奏、钩子机制等创作方法论
- 格式规范体系：剧本格式、场次标注、导出校验等标准化流程
- AI 协作工作流：从小说到剧本的完整流程指南

## 依赖关系

```
Phase 1 (经验沉淀) ──→ Phase 2 (架构升级) ──→ Phase 3 (Pipeline补齐) ──→ Phase 4 (质量提升) ──→ Phase 5 (多题材扩展)
```

严格线性依赖：Phase 1 的方法论指导后续设计，Phase 2 的架构是 Phase 3 的基础，Phase 3 完成后才能做质量控制，最后扩展题材。

## 风险提示

- Phase 1 方法论文档质量直接影响后续所有 Phase 的设计决策
- 全面切 Claude 后需验证各环节效果不低于原多模型方案
- Pipeline 补齐工作量较大（6个步骤），需注意控制范围
- 多题材扩展可能需要大量新样例数据支撑
