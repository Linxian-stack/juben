# Task05：多题材扩展

## 概要
将项目从特定题材扩展为通用框架：通用层 + 题材层分层模板，风格参数可配置，新样例一键接入。

## 依赖
- 前置：task04（质量控制体系作为通用基础）
- 后续：无（最终阶段）

## Phase 列表

| Phase | 文件 | 内容 | 产出物 |
|-------|------|------|--------|
| 01 | phase01.md | E1 题材模板体系 | `juben_gen/genres/` 目录 |
| 02 | phase02.md | E2 风格画像参数化 | `juben_gen/style_profile.py`（重构） |
| 03 | phase03.md | E3 新样例接入 | `juben_gen/cli.py`（扩展 profile 命令） |

## 进度追踪

- [x] phase01: E1 题材模板体系
- [x] phase02: E2 风格画像参数化
- [ ] phase03: E3 新样例接入

## 执行顺序
Phase 01 → 02 → 03（先定义模板结构，再参数化，最后工具化接入流程）

## 验收标准
- 通用层（红果风格）与题材层（末世/宫斗/甜宠/悬疑/穿越）分离
- 每个题材有独立 JSON 配置：特征 + 角色类型 + 冲突模式 + 标志性场景
- 通用指标统一，题材指标可配置
- 提供样例 DOCX → 自动统计 → 生成题材配置（至少1个样例）
